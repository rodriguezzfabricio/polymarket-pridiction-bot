# Polymarket Trading Bot Configuration

## Project Overview
Python FastAPI trading bot for Polymarket prediction markets.
Uses py-clob-client for order placement via CLOB API.

## Tech Stack
- Python 3.12
- FastAPI + Uvicorn
- Pydantic for data validation
- py-clob-client for Polymarket API
- httpx for async HTTP calls

## Key Commands
Start server: uvicorn src.main:app --reload
Run tests: pytest tests/ -v
Type check: mypy src/
Lint: ruff check .

## Code Style Rules
- Use async/await for ALL I/O operations
- Type hints on every function signature
- Pydantic models for all data validation
- Early returns for validation (avoid deep nesting)
- Explicit error handling with try/except
- Log every important action

## Project Architecture
src/
├── main.py              # FastAPI app entry
├── core/config.py       # Settings from .env
├── api/routes/          # FastAPI endpoints
├── services/            # Business logic
└── models/              # Pydantic schemas

## Trading Bot Rules (CRITICAL)
- NEVER exceed MAX_POSITION_SIZE from config
- ALWAYS check wallet balance before placing orders
- Log every trade decision with timestamp and reasoning
- Validate all market data before using it
- Handle API rate limits gracefully

## What NOT to do
- Don't hardcode API keys (use .env)
- Don't skip error handling on external API calls
- Don't write functions longer than 50 lines
- Don't commit .env files to git
